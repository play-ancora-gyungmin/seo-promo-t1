# BRIEF.md

## 프로젝트명
학원 강사 랜딩 템플릿 1차 구축 - 서하늘 영어

## 프로젝트 성격
이 프로젝트는 특정 강사의 홍보용 랜딩페이지를 만드는 동시에,
다른 강사/과목/브랜드에도 재사용 가능한 랜딩 템플릿 기준안을 만드는 작업이다.

즉 결과물은 아래 두 역할을 동시에 수행해야 한다.

1. 서하늘 영어 실제 랜딩페이지
2. 학원 강사 랜딩 템플릿의 첫 번째 기준 시안

---

## 배경

지금 필요한 것은 복잡한 서비스가 아니라,
짧은 시간 안에 완성도 있는 랜딩페이지를 만들고
그 구조를 나중에 다른 강사 버전으로 재사용할 수 있게 하는 것이다.

1차에서는 거대한 시스템보다
명확한 구조, 빠른 구현, 안정적인 템플릿화가 중요하다.

---

## 목표

- 서하늘 영어 랜딩페이지를 실제 배포 가능한 수준으로 제작한다.
- 같은 구조를 다른 강사 데이터에도 적용할 수 있게 한다.
- 학생과 학부모 모두에게 읽기 쉽고 신뢰감 있는 화면을 만든다.
- 향후 공지/칼럼/게시판성 페이지 1~2개를 붙일 수 있는 구조를 만든다.
- 이미지 최적화까지 포함한 실제 운영 가능한 기준안을 만든다.

## 타깃

### 0차 타깃

- 20대 중반 강사
- 자신의 강의를 마케팅할 목적
- 폰트 스타일을 신경쓰며 성과를 보기 좋게 PR하길 원함

### 1차 타깃

- 40대 전후 학부모
- 정보 구조, 신뢰도, 상담 편의성 중시
- 가독성과 접근성에 민감

### 2차 타깃

- 10대 학생
- 세련되고 답답하지 않은 화면 선호
- 강사의 인상과 분위기에 민감

## UI 전략

- 외부 완성형 디자인 라이브러리는 사용하지 않는다.
- Headless UI도 1차에서는 사용하지 않는다.
- Hero, Teacher Intro, Curriculum, Proof 같은 브랜드 섹션은 직접 구현한다.
- Button, Card, Input 같은 공통 UI만 Panda recipe 단위로 관리한다.

## 템플릿 전략

- 강사 이름, 과목, 카피, 후기, FAQ, 이미지 등은 데이터 파일로 분리한다.
- 같은 섹션 구조에 다른 강사 데이터를 넣어도 유지되게 한다.
- `enabledSections` 와 `sectionOrder` 로 일부 섹션 노출과 순서를 제어할 수 있게 한다.
- 현재 구현물은 “서하늘 영어 전용 페이지”가 아니라 “서하늘 영어 버전으로 구현된 템플릿”이어야 한다.

## 콘텐츠 스키마 역할

콘텐츠 스키마는 템플릿이 기대하는 데이터 규격이다.

목적:

- 데이터 구조 통일
- 컴포넌트 안정성 확보
- 템플릿 복제 용이성 확보
- 에이전트 작업 안정성 확보

---

## 이미지 전략

- 실제 콘텐츠 이미지와 강사 사진은 `src/assets`에 둔다.
- `public/` 사용은 최소화한다.
- Hero 대표 이미지는 우선 로딩 대상이다.
- 일반 이미지는 lazy loading을 유지한다.
- SVG로 가능한 장식과 아이콘은 SVG를 우선 사용한다.
- CLS 방지를 위해 크기 정보를 명확히 처리한다.
- 실제 렌더 크기에 맞는 이미지를 사용한다.

## 성공 기준

- 서하늘 영어 랜딩이 실제 서비스처럼 보인다.
- 다른 강사 데이터로 프리뷰가 가능하다.
- 학생과 학부모 모두 핵심 정보를 빠르게 이해할 수 있다.

- 구조가 특정 인물에게 묶이지 않는다.
- 이미지 최적화 원칙이 반영되어 있다.
- 향후 posts 페이지를 붙이기 쉬운 상태다.
